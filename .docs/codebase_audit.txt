eCodebase Audit: Prioritized Refactor Plan

  Phase 2 — High Priority

  5. Split quest_builder.py (776 LOC) (services/quest_builder.py)
  One file does 7 unrelated things: canvas rendering, 11-rule validation, form parsing, scene/choice CRUD, combat encounter management, requirements building. Split into: QuestCanvasRenderer, QuestValidator, QuestFormParser, CRUD functions. Each will be individually testable.

  6. Extract PlayerContext construction (services/scene.py:37-44, 70-81, and elsewhere)
  The same 4-line PlayerContext(stats, inventory, completed_map, flags) block appears 3+ times. A signature change requires hunting all call sites. Extract to build_player_context() in session.py.

  7. Create a session-loading decorator (views.py)
  Every view calls load_session_context(session_pk) and then repeats the if not session_pk: redirect('game_hub') guard. Extract to a @require_game_session decorator so the pattern lives in one place.

  8. Fix quest completion race condition (services/progression.py:77-85)
  The exists-then-create pattern has a TOCTOU window. Replace with CompletedQuest.objects.get_or_create(...) — the created flag tells you whether to award XP and append logs.

  ---
  Phase 3 — Refactor

  9. Clarify stat naming (constants.py, views.py)
  DB names (strength, agility) and display names (muscle, reflexes) are mixed across layers. Create explicit STAT_DISPLAY_NAMES = {'strength': 'muscle', ...} and STAT_DB_NAMES (its inverse) so the mapping direction is unambiguous.

  10. Strategy pattern for Requirement.evaluate() (models/requirements.py:62-94)
  A 30-line if-elif chain over 11 condition types. Adding a new type means modifying the class. Replace with an evaluator registry — new types are registered, not patched in.

  11. Collapse CombatState pending attack fields (models/combat.py:58-65)
  4 separate nullable fields (pending_e_roll, pending_e_damage, pending_e_mod_str, pending_e_hit) represent one optional object. Replace with a single pending_enemy_attack = JSONField(null=True).

  ---
  Phase 4 — Cleanup

  12. _htmx_response() hardcoded template list (views.py)
  Adding a 6th partial requires editing this function. Replace the 5-template string concatenation with a single base template that {% include %}s the partials, or drive the list from a setting.

  13. Add prefetch helper for quest requirements (services/scene.py, quest_builder.py)
  prefetch_related('requirements__requirements') appears in 3 places. Any call site that forgets it causes N+1 queries. Extract to a get_quests_with_requirements(queryset) helper.

  14. Docstrings on all public service functions
  Most functions in services/ have no docstring. Given the non-obvious parameter combinations (e.g. effective_stats vs stats), at minimum document args and return types.
